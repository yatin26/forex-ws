In this section, the methadology behind the prediction model used in this website to predict equity is explained below.

In the following model, we use classifiers to predict from the data whether the prices of different currency pairs will increase or decrease, by assigning it a label 1/0, to confirm the model using backtesting we allot equity and carry on trades from the data when the label is 1 and reject when the label reads 0.

The different currencies used in this model to form subsequent currency-pairs are:

  •  INR - Indian Rupee
  •  USD - United States Dollar
  •  GBP - British Pound Sterling
  •  EUR - Euro
  •  HKD - HongKong Dollar

EXTRACTING THE DATA

The data for creating the model was extracting from a Swiss bank called Dukascopy, it is a bank which deals with forex spot trading and provides services, it also has an extensive day to day dataset of exchange prices of all major currencies. The data of the currencies traded was mostly from the time period of 1st Jan 2010 to 31st December 2019, but this time-period was different for each currency-pair. These range from a period of 1 month to 10 years. Furthermore, the data was downloaded in a .csv format, easily accessible from MS Excel.

UNDERSTANDING THE DATA

Now, after having extracted the data we plotted the price of the Rupee to a single unit of the Dollar on a graph to just get a visual representation and subsequently a better understanding, Using matplotlib in python 3.7 in jupyter notebook(IDE). We plotted the everyday closing price of the currency pair.

MANIPULATING AND ADDING ROWS

After rudimentary plotting, we create another column parameter ‘RETURN’ which is simply the price difference between two consecutive days. We create another variable return range which is the maximum return subtracted from the minimum return.

SHAPING AND TRAINING THE DATA

We create another column ‘LABEL’ which is simply, the value of return, if return is +ve then label is 1 else 0. We train 95% of the data and test 5% in the model.

GRADIENT BOOST CLASSIFIER

They are various ML algorithms combined together, which are weak in an augmented manner to create a strong predictive model. Features here are the inputs for the ML algorithm which are used to calculate the output.

PROFIT FUNCTION

The profit function is created using a pertinent strategy in finance called backtesting. We first create new parameters such as ‘pred’, ‘won’ and ‘return’, where we link the data from the column ‘label’ to our predicted model’s ‘label’. The won parameter will be true only if both the ‘pred’ and ‘label’ parameters were the same.

We then converted the python code obtained for the prediction model into javascript with the help of some open source tools and added that to our website created by the help of HTML and CSS.
